Contribute Templates

Contribute Your Template

Help grow the NeuroSites community by contributing your own website templates! We welcome contributions from researchers, developers, and designers who want to share their work with the neuroscience community.

Ready to contribute?

Start by forking our repository and reading our detailed contributing guidelines.

Fork Repository

Template Requirements

To ensure quality and consistency, all template submissions must meet these requirements:

Technical Requirements

  • Clean HTML/CSS: Well-structured, semantic HTML with organized CSS
  • Responsive Design: Works well on desktop, tablet, and mobile devices
  • Cross-browser Compatibility: Functions properly in modern browsers
  • Performance: Optimized images and efficient code
  • Accessibility: Follows basic accessibility best practices

Content Requirements

  • Relevance: Suitable for neuroscience or academic communities
  • Professional Quality: Clean, polished design appropriate for research contexts
  • Placeholder Content: Generic content that users can easily customize
  • No Personal Information: Remove any personal data before submission

Documentation Requirements

  • README.md: Clear instructions for customization and deployment
  • Preview Image: preview.png (recommended 1200x800px)
  • Manifest File: manifest.json with template metadata
  • License: Must be compatible with CC BY 4.0

Submission Process

Step 1: Prepare Your Template

  1. Create a clean, well-organized template
  2. Test it thoroughly across different devices and browsers
  3. Remove any personal information or proprietary content
  4. Write clear documentation

Step 2: Fork and Clone

  1. Fork the NeuroSites repository
  2. Clone your fork to your local machine
  3. Create a new branch for your template

Step 3: Add Your Template

  1. Navigate to the appropriate category folder in /templates/
  2. Create a new folder for your template (use kebab-case naming)
  3. Add all your template files
  4. Include the required documentation files

Step 4: Submit Pull Request

  1. Commit your changes with a clear, descriptive message
  2. Push to your fork
  3. Submit a pull request with a detailed description
  4. Our team will review and provide feedback

Template Categories

We organize templates into these main categories:

  • Lab Websites: Research group homepages, project sites
  • Personal Pages: Academic portfolios, researcher profiles
  • Event Promotions: Conference sites, workshop pages
  • Package Wikis: Software documentation, package sites

If your template doesn't fit these categories, suggest a new one in your pull request!

Manifest File Format

Include a manifest.json file with your template containing:

{
  "title": "Your Template Name",
  "description": "Brief description of the template",
  "category": "Lab Websites",
  "tags": ["modern", "research", "responsive"],
  "style": "Modern",
  "theme": "Blue",
  "author": "Your Name",
  "license": "CC BY 4.0",
  "version": "1.0.0"
}

Review Process

Once you submit a pull request, our team will:

  1. Initial Review: Check that all requirements are met
  2. Technical Review: Test functionality and responsiveness
  3. Quality Review: Assess design quality and code organization
  4. Feedback: Provide suggestions for improvements if needed
  5. Approval: Merge approved templates into the main repository

Recognition

Contributors are recognized in several ways:

  • Listed in the template's metadata and README
  • Added to our contributors list
  • Acknowledged in release notes
  • Invited to join our maintainer team (for regular contributors)

Questions?

If you have questions about contributing, please:

  • Check our detailed contributing guide
  • Browse existing discussions
  • Start a new discussion or submit an issue

We're excited to see what you'll create! Thank you for helping make NeuroSites a valuable resource for the entire community.
